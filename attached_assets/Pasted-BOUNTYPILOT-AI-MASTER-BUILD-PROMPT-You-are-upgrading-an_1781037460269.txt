PRODUCT VISION

BountyPilot AI is not a bounty tracker.

It is a personalized autopilot agent for creator revenue workflows.

The goal is to help creators move from:

Opportunity Discovery → Opportunity Evaluation → Research → Content Planning → Submission → Earnings Tracking

with as little manual effort as possible.

The system should feel like an intelligent creator operating system rather than a dashboard.

⸻

CORE USER STORY

A creator discovers a bounty opportunity.

Instead of manually researching everything, they paste the bounty URL into BountyPilot AI.

The agent:

1. Reads the page
2. Extracts important information
3. Detects missing information
4. Attempts fallback extraction
5. Scores the opportunity
6. Explains the reasoning
7. Requests human approval
8. Generates a research brief
9. Generates a production plan
10. Tracks submission
11. Tracks earnings

⸻

REPLIT CORE REQUIREMENTS

This application is being built and deployed on Replit Core.

Use Replit Secrets / API Manager for all credentials.

Never hardcode secrets.

Read secrets from Replit first.

Required secrets:

* QWEN_API_KEY
* QWEN_BASE_URL
* QWEN_MODEL
* LLM_PROVIDER

Supported providers:

* mock
* qwen

Behavior:

If QWEN_API_KEY exists:

* Use Qwen provider.

If QWEN_API_KEY does not exist:

* Automatically fall back to mock mode.
* Show a warning banner.
* Continue functioning normally.

The app must never crash because of a missing API key.
